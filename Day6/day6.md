# Day 6 – Resume Optimization & ATS Analysis

## Objective

Learn how AI can improve a resume for Applicant Tracking Systems (ATS), recruiter readability, and professional branding while keeping all information truthful.

---

# ATS Analysis

## ATS Score Comparison

| Version          | Score    |
| ---------------- | -------- |
| Original Resume  | 52 / 100 |
| Optimized Resume | 88 / 100 |

### Improvement

Score Increased: **+36 Points**

---

# What Changed and Why It Improved the Score

## 1. Added Professional Summary

### Original Resume

Contained a generic objective statement.

### Optimized Resume

Added a professional summary highlighting:

* MCA candidate
* Full-stack development experience
* AI-powered healthcare project
* Database and analytics skills

### Impact

Recruiters and ATS systems expect a targeted summary section.

---

## 2. Strong Action Verbs

### Before

* Developed
* To handle
* For tracking

### After

* Designed
* Developed
* Built
* Integrated
* Delivered

### Impact

Makes achievements more measurable and impactful.

---

## 3. ATS-Friendly Section Headings

### Original

* Objective
* Technical Skills
* Certifications

### Optimized

* Professional Summary
* Technical Skills
* Certifications

### Impact

ATS systems parse standard headings more accurately.

---

## 4. Better Technology Visibility

Additional keywords were highlighted:

* Flask
* Spring Boot
* NumPy
* Pandas
* Matplotlib
* Bootstrap
* MySQL

### Impact

Improves keyword matching against job descriptions.

---

## 5. Improved Project Descriptions

### Original

Project descriptions were brief and less achievement-focused.

### Optimized

Projects clearly describe:

* Problem solved
* Technologies used
* Features implemented
* Business value delivered

### Impact

Improves recruiter readability.

---

## 6. Cleaner Formatting

### Original Resume

* Multiple formatting styles
* Less consistent structure

### Optimized Resume

* Single-column format
* ATS-safe structure
* Easy to scan

### Impact

Improves ATS parsing and recruiter experience.

---

# Resume Comparison

## Original Resume Highlights

### Strengths

* Good academic background
* Relevant projects
* Multiple certifications

### Weaknesses

* Generic objective
* Limited project impact descriptions
* Fewer ATS keywords
* Less professional branding

---

## Optimized Resume Highlights

### Strengths

* Professional Summary
* Strong action verbs
* Better keyword coverage
* Clear project achievements
* ATS-safe formatting
* Improved recruiter readability

---

# Key Learnings

1. ATS systems prefer standard section headings.
2. Professional summaries improve resume quality.
3. Strong action verbs increase impact.
4. Relevant keywords improve ATS matching.
5. Simple formatting is often better than complex designs.
6. Recruiters spend only a few seconds reviewing a resume.

# Conclusion

This exercise demonstrated how AI can significantly improve resume quality without changing the truthfulness of the content. By adding a professional summary, improving keyword optimization, strengthening project descriptions, and using ATS-friendly formatting, the resume score increased from 52/100 to 88/100, making it more competitive for internships and entry-level software engineering roles.
