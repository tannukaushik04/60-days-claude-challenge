# Day 51 – Product Discovery & Sprint Planning

## 📌 Objective
For Day 51 of the AB Talks 60-Day Claude AI Challenge, I focused on the product discovery and planning phase of my 10-day capstone project. Instead of jumping directly into coding, I worked on defining the problem, refining the project scope, preparing a Product Requirements Document (PRD), creating a complete implementation blueprint, and designing a project pitch deck that will guide the remaining development days.

---

# 🚀 Final Capstone Project

## Project Name
**WhyIFeel**

### Tagline
**Understand what actually makes you feel better.**

WhyIFeel is a habit and mood tracking application that helps users discover how their daily habits affect their energy, mood, stress, and soreness using a self-built statistical insight engine. Instead of relying on generic AI advice, it analyzes the user's own historical data and generates personalized insights with confidence levels.

---

# 📄 Deliverables

## ✅ Product Requirements Document (PRD)

Created a professional PRD covering:

- Problem Statement
- Product Vision
- Target Users
- Functional Requirements
- Non-functional Requirements
- User Flows
- Data Model
- Insight Engine
- Risks
- Assumptions
- Scope Definition

### Highlights

- Username/password authentication
- Daily habit and feeling tracking
- Personalized statistical insight engine
- Interactive dashboard
- Demo data generator
- Free-tier deployment target

---

## 🛠️ Implementation Blueprint

Prepared a complete **Days 2–10 implementation roadmap** containing:

- Daily objectives
- Features to build
- Folder structure
- Files to create
- APIs/Libraries
- Testing strategy
- Debugging tips
- End-of-day checklist
- Screenshot checklist
- Handoff notes

This blueprint serves as the single source of truth for the remainder of the capstone.

---

## 📊 Project Pitch Deck

Prepared a presentation-ready pitch deck covering:

- Problem
- Target Users
- Solution
- Key Features
- Technical Approach
- Future Scope
- Product Vision

The deck explains how WhyIFeel transforms simple daily habit logs into meaningful, confidence-based insights using a custom statistical engine.

---

# 🎯 Finalized v1.0 Scope

## Included

- User authentication
- Daily check-in (4 habits + 4 feelings)
- One entry per day
- Statistical insight engine
- Confidence-based insights
- Dashboard with charts
- Date range filtering
- Demo data generator
- Deployment on a free hosting platform

## Excluded (Future Versions)

- Editing past entries
- Password reset
- Email verification
- OAuth login
- Mobile app
- Push notifications
- Wearable integration
- External AI APIs
- Social features
