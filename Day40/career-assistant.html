System Prompt Design

The assistant runs a single, unified system prompt with four distinct modes (resume review, mock interview, job search strategy, LinkedIn branding), selected via a mode tag prepended to each request. This keeps one coherent "brain" while letting the UI route to specialized behavior — role, scope, and output contract are shared; only the task-specific instructions and output schema change per mode.

  • Role & scope: explicitly defined as a professional career coach, with clear boundaries against legal/HR advice, discriminatory content, or fabricating facts about the user.
  • Output format: the model is instructed to return a structured "report" using lightweight markdown-like headers (##) and bullets that the frontend parses into styled sections, scores, and verdict badges — plus plain conversational text for follow-up chat turns.
  • Edge cases: handles missing/thin input (asks for what's needed instead of hallucinating), irrelevant or hostile input (redirects politely), and overly long pasted documents (summarizes key issues rather than repeating content back).

UI Decisions

  • Four-mode navigation mirrors the four use cases surfaced in discovery, so users self-select the right workflow instead of guessing what to type.
  • Mixed input model (form fields + free text + file upload) matches how people actually arrive: some have a job description handy, some just want to paste a resume, others want to type from scratch.
  • Structured report → follow-up chat pattern: the first response is a scannable, visual report (score badge, verdict tag, headers); after that, the conversation drops into a normal chat thread so users can ask "why" or request rewrites without re-submitting the whole form.
  • Client-side only: no build step, no external libraries — a single HTML file that calls the Anthropic Messages API directly via fetch, with the API key stored only in memory/localStorage on the user's own machine.

Future Extensions

  • Tools: add a job-description-fetch tool (URL → parsed JD) or a real ATS-keyword-matching tool for more precise resume scoring.
  • Memory: persist a user's resume/profile across sessions so mock interviews and strategy sessions can reference prior context automatically.
  • Multi-step workflows: chain modes — e.g., resume review → auto-generate matching cover letter → auto-generate likely interview questions for that same role — as a guided pipeline rather than four separate tools.
  • Export: one-click export of the report or rewritten resume/cover letter to a downloadable document.
