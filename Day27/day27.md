# Day 27 – Prior Authorization Story Simulator

## 📌 Project Overview

I built a **Prior Authorization Story Simulator**, an interactive healthcare education application that explains the **U.S. Prior Authorization (PA)** process through storytelling instead of traditional documentation.

The simulator follows **Rahul**, a patient diagnosed with Rheumatoid Arthritis, and **Priya**, a healthcare operations specialist, who guides the learner through every stage of the Prior Authorization journey using simple conversations and branching choices.

The application is designed to make a complex healthcare workflow easier to understand through an engaging, beginner-friendly experience.

---

# 🚀 Project Objectives

- Learn healthcare workflows using storytelling.
- Understand the complete Prior Authorization lifecycle.
- Build an interactive browser application using HTML, Tailwind CSS, and Vanilla JavaScript.
- Create a conversational UI with branching decisions.
- Explain complex insurance concepts in simple language.

---

# 🛠 Technologies Used

- HTML5
- Tailwind CSS (CDN)
- Vanilla JavaScript
- DOM Manipulation using `createElement()` and `appendChild()`
- Responsive UI Design

---

# 📖 Story Flow

The simulator contains **8 interactive scenes**:

1. Doctor Visit
2. Insurance Roadblock
3. What is Prior Authorization?
4. Insurance Review
5. Denial
6. Appeal
7. Approval
8. Final Takeaways

Each scene presents:

- Educational conversations
- Story narration
- Interactive decision buttons
- Progress tracking
- Beginner-friendly explanations

# 💡 Key Takeaways from Rahul's Journey

### 👨 Patient Perspective

- Prior Authorization is **not** a medical decision—it is an insurance review process.
- A denial does **not** always mean treatment is rejected permanently.
- Missing clinical documentation is one of the most common causes of delays.
- Asking questions early helps patients understand the process better.
- Keeping approval reference numbers is important for future treatments.

---

### 🏥 Healthcare Operations Perspective

The simulator demonstrates how healthcare providers and insurers manage Prior Authorization by reviewing:

- Insurance eligibility
- Clinical documentation
- ICD-10 diagnosis
- Medical necessity
- Step therapy history
- Appeals
- Final approval

It also highlights operational metrics such as:

- Denial Rate
- Appeal Rate
- Resolution Time
